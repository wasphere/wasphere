# WaSphere v1.1 — Inbox Test Plan

Verification plan for the v1.1 Inbox feature (PR #115). Three layers: **automated**
(real Postgres), **build**, and **manual real-WhatsApp**. This anchors what
"v1.1 complete" means and serves as the v1.2 regression baseline.

> Testing rule (CLAUDE.md): no mock tests for WhatsApp behaviour — automated tests
> run against a real PostgreSQL (Docker); WhatsApp flows are verified manually with
> a real test number.

---

## A. Automated tests (real PostgreSQL, no mocks)

```bash
cd packages/dashboard-api
pnpm test:setup     # once — creates + migrates the wasphere_test DB (Docker)
pnpm test           # builds, then runs the integration suite
```

**Expected:** `tests 13 · pass 13 · fail 0`.

Coverage:

| Area | Cases |
|---|---|
| Ingestion | LID→real phone (senderPn); group/newsletter/status/broadcast filtered; idempotent dedup; undecryptable→decoded **upgrade**; media stored in `mediaUrl` (not payload); unread increment; archive on logout + restore on reconnect; stale flag cleared by new message |
| Service | non-member rejected; search by name + **non-match regression** + phone; markRead; patch (status/tags/notes); status filter |

## B. Build verification

```bash
cd packages/wa-server   && pnpm build   # ✓ tsc
cd ../dashboard-api     && pnpm build    # ✓ nest/tsc
cd ../dashboard-ui      && pnpm build    # ✓ next build (type-check + static pages)
```

## C. Stack startup

```
Postgres (Docker) → wa-server :3001 → dashboard-api :3300 → dashboard-ui :3304
```

Open `http://localhost:3304/dashboard/inbox`, log in, **Sessions → link** a real
WhatsApp (scan QR). Local-dev env quirks: repo-root `.npmrc` must set
`node-linker=hoisted` (Prisma custom output); `DASHBOARD_INTERNAL_URL` must point
the wa-server callback at the local API; `INTERNAL_WEBHOOK_SECRET` must match
between wa-server and dashboard-api.

## D. Manual E2E checklist (two phones: linked number + a "customer")

**Receive (customer → you)**
- [ ] Text → appears live, correct **name + real number** + **photo**
- [ ] Image → inline + lightbox + **Download**
- [ ] Voice note / audio → plays · Video → plays + **full-view** lightbox · Document → downloads
- [ ] Sticker → renders
- [ ] Unread badge increments; opening clears it
- [ ] Reaction from customer → "reacted 🙂" line

**Send (you → customer)**
- [ ] Text (Enter sends) · Quick replies (+ Manage quick replies)
- [ ] 📎 → Image / Document / **Poll** (single **and** multi-select)
- [ ] React (hover/tap → emoji bar + full picker via "+")
- [ ] Forward (⋮ → Forward → pick chat) · Copy
- [ ] Sent image shows the real picture

**Polls (Shopify-critical)**
- [ ] Customer votes → **"🗳️ Voted: …"** (no "couldn't be loaded")
- [ ] Customer changes vote → updates cleanly
- [ ] Webhook: subscribe a URL to `poll.vote` (or `message.received`) → receives `selectedOptions` + `voter.phone`

**Inbox ops**
- [ ] Tags add/remove · Notes save · Mute toggle · Media & docs gallery
- [ ] Resolve / Reopen (⋮, moves between tabs)
- [ ] Search by name (only matches), phone, preview
- [ ] Session filter — "All sessions" vs a specific session
- [ ] Realtime — new message appears without refresh; "live" badge green
- [ ] Sound — beep on inbound when unfocused (unless muted)

**Edge cases (hard-won)**
- [ ] Logout session → re-link → chat un-archives, can send again
- [ ] Group / status / channel messages → do **not** appear (1:1 only)
- [ ] Album/media send → no stray "associatedChildMessage" bubble
- [ ] Mobile → list/thread switch, **Back** button, only chat scrolls, react/forward buttons tappable, contact-info sheet opens

## E. Known limits (deferred to v1.2 — not failures)

- Media > 16 MB skipped inline (streaming endpoint is the v1.2 fix)
- Location → "📍 Location" label (no map preview)
- Contact card (vCard) → "👤 Contact" label (no parsed card)
- Poll re-voting can't be locked at the WhatsApp level (handle via grace-window +
  delete-to-lock in the integration layer)

---

**Sign-off:** v1.1 is considered verified when A is green, B compiles, and the D
checklist passes against a real WhatsApp number.
